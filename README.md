# Анализ рынка IT-вакансий для образовательной платформы

## 1. О проекте

Данный проект посвящен анализу рынка IT-вакансий для обновления образовательных программ. На основе собранных данных мы исследовали, какие навыки и требования чаще всего встречаются у работодателей, какие из них связаны с более высокой зарплатой, и какие темы стоит усиливать в учебной программе, чтобы обучение лучше соответствовало актуальному спросу рынка. Со стороны заказчика рассматривается образовательная платформа (Яндекс Практикум, Skillbox, Нетология или т.д), которая и хочет понять, что сейчас актуально среди работодателей, для составления актуальных курсов:

### Бизнес-проблема
Рынок IT меняется быстро, а учебные программы обновляются медленнее, т.е могут наблюдаться высокие риски, что выпускники в таком случае получают неактуальные знания,
спрос на новые инструменты и роли на рынке, которые не покрываются программой рассматриваемой платформы, растет, выпускники получают меньше приглашений на собеседования и офферы ниже рынка, как следствие снижается ценность продукта, репутация и конверсия в повторные покупки

Поэтому мы решили рассмотреть:
Чему нужно учить прямо сейчас, чтобы программа обучения соответствовала актуальному спросу рынка?

### Практический результат для заказчика
По итогам анализа заказчик получает:
- список наиболее востребованных навыков и знаний
- навыки и технологии, которые чаще всего встречаются в вакансиях с высокой зарплатой
- понимание того, какие темы нужно включить в базовый трек, а какие в продвинутый
- рекомендации по обновлению учебной программы

## 2. Источник данных для скраппинга

В качестве источника был выбран сайт HH.ru. С помощью фильтров, в нашем случае использовался фильтр по "Специализации", были отобраны актуальные для нас профессии, почему использовался фильтр: максимальное количество страниц доступных к просмотру 40, на каждой странице располагается 50 вакансий, т.е всего для парсинга было доступно 2000 вакансий из примерно 19500 вакансий, поэтому было принято решение фильтровать по нескольким вакансиям, чтобы получить необходимый по размеру датасет

### По итогу было собрано 5 файлов:
- hh.csv
- hh2.csv
- hh3.csv
- hh4.csv
- hh5.csv
Каждый из этих файлов содержит по 2000 строк, в итоге после объединения получен итоговый файл dd_hh.csv размером 10 000 строк, как требовалось по условию, после этого данные были приведены к единому виду.

### Единица наблюдения
Одна строка = одна вакансия, с информацией содержащейся на ее странице

### Период публикации вакансий в загруженном датасете
Вакансии охватывают период с 2025-07-09 по 2026-04-10

## 3. Подробное описание данных

По работы с объединенными данными был сформирован df.csv, в него, помимо исходных данных, были добавлены производные зарплатные признаки, а также бинарная целевая переменная для определения высокой зарплаты

### Размер набора данных

4009 объектов и 12 признаков (сокращение количества обьектов произошло всвязи с удаление дубликатов, очисткой NaN значений и данных, которые не подходили для анализа)

P.s. в final_eda остается лишь 2470 обьектов, так как было принято решение взять данные с Января 2025 года

### Что является объектом наблюдения

Один объект - вакансия, для которой сохранены:

- источник публикации
- дата
- полный текст вакансии
- зарплатные признаки
- метка высокой зарплаты
- структурированные признаки: грейд, формат работы, технологии, специализация
